Build a complete AI Resume Builder web application from scratch.
Tech Stack:
  • Frontend: React.js + Custom CSS + Lucide Icons
  • Backend: Java 17+, Spring Boot 3.x (Spring Web, Spring Data JPA, Spring Security)
  • Database: MySQL
  • AI: Groq API - Llama-3.3-70b-versatile (Modular architecture to allow transition to Gemini or Claude)
Project Objective:

Develop an AI-powered resume builder platform that dynamically extracts user details through interactive conversations, generates an ATS-optimized, professionally formatted resume, and allows real-time layout and style customization.

User Inputs:
  • Initial User Prompt / Career History
  • Answers to targeted follow-up questions
  • Live Content Editor fields (Full Name, Summary)
  • Styling presets (Modern, Classic, Compact)
  • Layout controls (Margin, Line Height, Text Size)
Functional Requirements:
  1. Dynamic Question Generation: Evaluate the initial user prompt and dynamically generate follow-up questions based on the target role, experience level, and key skills to gather missing components.
  2. Persona-Specific Questions: Ensure different job roles receive tailored, highly professional questions. Avoid using generic or repetitive questions.
  3. Pre-Generation Verification: Run the complete information gathering state machine before initiating the final resume layout.
  4. Interactive Progress Flow: Present one question at a time or display targeted follow-up groups with loading states.
  5. Intelligent Parsing: Automatically extract and organize structured segments (Summary, Experience, Projects, Education) from natural conversational text.
  6. Database Integration: Persist user profiles, resume states, experiences, and configurations in a structured MySQL schema.
  7. Deferred Evaluation: Do not submit half-formed sections to the AI. Wait for all gathered answers to be submitted collectively to generate the comprehensive layout.
  8. Real-Time Layout Engine: Bind margins, line height, font faces, and color themes directly to CSS variables for instant style updates.
  9. Clean Export Pipelines: Provide clean, print-ready CSS print queries to export the finished resume as a high-quality A4 PDF without showing UI/workspace components.
Final Resume Output should include:
  • Professional Summary (ATS-optimized with strong industry action verbs)
  • Work Experience (Company, Role, Dates, and Bullet-point Achievements)
  • Projects (Project Name, Links, and Core Description)
  • Education (Institution, Degree, Dates, and Major field of study)
  • Custom Style Configuration (Color themes, Font styles, Spacing metrics)
Application Requirements:
  • Modern, premium, responsive 50/50 split-screen layout
  • Smooth interaction flow with intuitive user prompt states
  • Active loading indicators while communicating with the AI service
  • Robust exception handling displaying actual API exception logs for quick debugging
  • Modular React components separated cleanly (Workspace, Panels, Canvas)
  • Clean, layered Spring Boot architecture (Controller, Service, Repository, Entity, DTO)
  • Environment variables for API keys to maintain code security
  • Well-structured, self-documenting, and maintainable codebase
Important Rules:
  • Do not use hardcoded resumes or mock data for final outputs.
  • Every resume must be personalized based on the specific user prompts.
  • The AI must act as a career coach, refining raw user descriptions into impact-driven bullet points.
  • The print layout must scale perfectly to standard A4 page dimensions without showing horizontal/vertical cutoffs.
  • The backend must implement proper dependency injection and modular service wrappers so that AI services (like Groq) can be hot-swapped for Gemini or Claude easily.
Deliver a complete, production-ready application with clean architecture, reusable components, and best coding practices.
